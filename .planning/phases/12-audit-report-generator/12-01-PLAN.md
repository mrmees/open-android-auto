    **scanner.py — `scan_audit_tree(repo_root: Path) -> ScanResult`:**

    1. `oaa_root = repo_root / "oaa"`
    2. Discover all subdirectories under `oaa_root` (19 currently: audio, av, bluetooth, carcontrol, common, control, generic, input, media, mediabrowser, mic, navigation, notification, phone, radio, sensor, verification, video, wifi)
    3. For each directory:
       - `glob("*.proto")` -> set of proto stems
       - `glob("*.audit.yaml")` -> set of sidecar stems
       - Missing = proto stems without matching sidecar stem
       - Orphan = sidecar stems without matching proto stem (strip `.audit.yaml` to get base name, check if `{base}.proto` exists)
       - For each sidecar: `yaml.safe_load`, extract `confidence` field -> increment ChannelCounts
       - For each sidecar: iterate `evidence[]` list entries, extract `type` field -> increment evidence_by_tier[tier][type]
       - **CRITICAL:** Do NOT iterate `pending_platinum_evidence[]` entries — skip them entirely for evidence counting
       - Check `oem_match_pending_gold: true` -> increment pending_gold_count
    4. Return ScanResult with all aggregated data

    **Naming convention for sidecar-to-proto matching:**
    - Sidecar `FooMessage.audit.yaml` matches proto `FooMessage.proto`
    - Strip `.audit.yaml` suffix, append `.proto`

    **Evidence type discovery is dynamic:** Use a `Counter` or `defaultdict(int)` keyed by whatever string appears in `evidence[].type`. No hardcoded list. If a sidecar has `type: "some_new_type"` it gets counted.

    **report.py — `render_markdown(result: ScanResult) -> str`:**

    Generate a markdown string with 6 sections in locked order:

    1. **Summary** — one-line headline: "{total_sidecars} sidecars covering {total_protos} protos ({coverage_pct}% coverage). {platinum} Platinum / single-OEM, {gold} Gold, {silver} Silver, {bronze} Bronze."
       - Coverage pct = `total_sidecars / total_protos * 100`, rounded to nearest int
       - If pending_gold_count > 0, add: "{pending_gold_count} protos awaiting deep-trace for Platinum promotion (oem_match_pending_gold)."

    2. **Per-Channel Tier Counts** — markdown table with locked columns:
       `| Channel | Bronze | Silver | Gold | Platinum (s-OEM) | Retracted | Superseded | Total |`
       - One row per directory (sorted alphabetically)
       - Total column = sum of all tier columns for that row
       - Summary row at bottom with totals

    3. **Evidence Type Breakdown** — markdown table:
       `| Tier | {dynamic columns from discovered evidence types} | Total |`
       - Rows = tiers (Bronze, Silver, Gold, Platinum, Retracted, Superseded) in that order
       - Columns = evidence types discovered from data, sorted alphabetically
       - Omit columns where ALL tiers have zero for that type
       - Cell = count of evidence entries of that type at that tier
       - Total column = sum across all evidence types for that tier

    4. **Missing Sidecars** — grouped by directory:
       ```
       ### audio (3 missing)
       - FooMessage.proto
       - BarData.proto
       - BazEnum.proto
       ```
       - Show total at section header: "85 proto files without audit sidecars"
       - If 0 missing: "All proto files have audit sidecars."

    5. **Orphan Sidecars** — same format as missing. Currently 0.
       - If 0: "No orphan sidecars found."

    6. **Dashboard Metadata** — run date, tool version ("1.0.0"), total proto count, total sidecar count, sidecar directory, git HEAD hash (from `git rev-parse HEAD`).

    **report.py — `render_json(result: ScanResult) -> str`:**

    JSON with top-level keys: `summary`, `per_channel_tier_counts`, `evidence_type_breakdown`, `missing_sidecars`, `orphan_sidecars`, `metadata`.
    - `summary`: {total_sidecars, total_protos, coverage_pct, tiers: {bronze, silver, gold, platinum, retracted, superseded}, pending_gold_count}
    - `per_channel_tier_counts`: dict of channel -> {bronze, silver, gold, platinum, retracted, superseded, total}
    - `evidence_type_breakdown`: dict of tier -> {type: count, ...}
    - `missing_sidecars`: dict of dir -> [proto_path, ...]
    - `orphan_sidecars`: dict of dir -> [sidecar_path, ...]
    - `metadata`: {run_date, tool_version, total_protos, total_sidecars, sidecar_directory, git_head}
    - `json.dumps(data, indent=2, sort_keys=True)` for deterministic output

    **run.py — `main(argv=None) -> int`:**

    1. `argparse` with:
       - `--output-dir` (default: `analysis/reports/coverage-dashboard/`)
       - `--quiet` (suppress stdout)
       - `--json-only` (no markdown)
    2. Determine `repo_root` as `Path(__file__).resolve().parents[3]` (the repo root containing `oaa/`)
    3. Call `scan_audit_tree(repo_root)` -> ScanResult
    4. Call `render_markdown(result)` -> md_str (unless --json-only)
    5. Call `render_json(result)` -> json_str
    6. Create output_dir, write files
    7. If not --quiet: print the per-channel tier table (just section 2 from markdown, not the full report) to stdout
    8. Print "Wrote {output_dir}/coverage-dashboard.md" and "Wrote {output_dir}/coverage-dashboard.json" to stderr
    9. Return 0
